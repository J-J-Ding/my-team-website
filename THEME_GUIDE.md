## 功能概述

智能系统实验室网站支持个性化主题和背景设置：

### 🎨 主题风格
- 6种预设主题风格
- 实时切换，无需刷新
- 支持"设为默认"，下次登录自动应用

### 🖼️ 背景设置
- 4种预设背景（渐变、图片、纯色）
- 成员可上传自定义背景（仅对当前用户可见）
- 游客可浏览所有预设背景

### 🔗 超链接功能
- 所有内容支持添加超链接
- 自定义链接文字
- 增强网站可读性和访问便利性

## 游客用户使用

### 1. 访问网站
直接打开网站首页

### 2. 切换主题
1. 点击右上角的 🎨 图标按钮
2. 在弹出的"主题风格"中选择喜欢的主题
3. 主题立即生效，无需刷新

### 3. 更换背景
1. 点击 🎨 图标打开设置面板
2. 在"背景设置"中选择喜欢的背景
3. 支持的背景类型：
   - 渐变背景（系统渐变）
   - 图片背景（预设图片）
   - 纯色背景（简约风格）

### 4. 保存设置
点击"保存设置"后，主题和背景选择会保存到本地缓存

**注意**：游客用户的设置仅保存在浏览器中，清除浏览器数据后会恢复默认。

## 成员用户使用

### 1. 登录账号
1. 访问 `pages/login.html`
2. 输入用户名和密码登录
3. 登录后进入个人中心

### 2. 访问主页
点击导航栏"主页"或 Logo 返回主页

### 3. 切换主题（同游客）
1. 点击右上角 🎨 图标
2. 选择喜欢的主题
3. 点击"保存设置"

### 4. 设置默认主题
1. 选择喜欢的主题和背景
2. 点击"保存设置"
3. 下次登录时自动应用

### 5. 上传自定义背景（成员专属）

#### 步骤
1. 点击 🎨 图标打开设置
2. 滚动到"自定义背景"区域
3. 点击"📁 上传背景图片"
4. 选择本地图片文件
5. 背景立即应用到 Hero 区域

#### 限制
- 支持的格式：JPG、PNG、GIF
- 最大文件大小：2MB
- 仅对当前登录用户生效
- 其他用户不会看到您的自定义背景

#### 清除自定义背景
1. 点击"🗑️ 清除自定义背景"按钮
2. 恢复为选择的预设背景

### 6. 恢复默认设置
点击"恢复默认"按钮，将重置为系统默认主题和背景

## 超链接功能

### 概述
所有上传的内容（项目、论文、竞赛、刊物、博客等）都可以添加超链接，让访问者能够快速跳转到相关页面。

### 使用方法

#### 在管理后台添加链接
1. 登录账号
2. 进入管理后台
3. 选择要编辑的内容类型
4. 填写以下信息：
   - `hasLink`: 是否启用链接（true/false）
   - `linkUrl`: 链接地址
   - `linkText`: 链接显示文字

#### 示例
```json
{
  "title": "智能推荐系统",
  "hasLink": true,
  "linkUrl": "https://github.com/your-repo",
  "linkText": "查看项目详情"
}
```

### 链接类型

#### 1. 项目链接
- GitHub 仓库链接
- 项目演示链接
- 项目文档链接

#### 2. 论文链接
- DOI 链接
- ArXiv 链接
- PDF 下载链接

#### 3. 竞赛链接
- 比赛官网链接
- 获奖证书链接
- 参赛作品链接

#### 4. 刊物链接
- 出版社购买链接
- 电子版下载链接
- 试读章节链接

#### 5. 博客链接
- 系统演示链接
- 相关文档链接
- 扩展阅读链接

### 最佳实践

1. **链接文字清晰**
   - 使用简洁明确的文字
   - 例如："查看论文"、"GitHub"、"项目演示"

2. **链接可访问性**
   - 确保链接有效且可访问
   - 使用绝对路径
   - 外部链接使用 `target="_blank"`

3. **链接数量控制**
   - 每个内容最多1-2个链接
   - 避免链接过多造成混乱

4. **链接样式统一**
   - 使用统一的链接样式
   - 保持视觉一致性

## 技术说明

### 数据存储
- **主题设置**：`localStorage` 浏览器本地存储
- **背景设置**：`localStorage` 浏览器本地存储
- **自定义背景**：Base64 编码存储在 `localStorage`

### 自定义背景存储
- 图片转换为 Base64 编码
- 存储键：`customBackground`
- 存储限制：约 2MB

### 默认设置
- 默认主题：`data/themes.json` 中的 `defaultTheme`
- 默认背景：`data/themes.json` 中的 `defaultBackground`

### 主题切换原理
1. 读取 CSS 变量
2. 动态修改 `--primary-color` 等变量
3. 所有引用该变量的元素自动更新

### 深色模式
- 深色主题自动添加 `.dark-mode` 类
- 使用 CSS 选择器覆盖样式
- 实现整体颜色翻转

## 常见问题

### Q1: 自定义背景为什么别人看不到？
A: 自定义背景仅对当前登录用户生效，其他用户看不到。这是设计特性，保护个人隐私。

### Q2: 如何分享我的主题设置？
A: 目前不支持分享设置。每个用户需要单独选择自己喜好的主题。

### Q3: 清除浏览器缓存后设置会丢失吗？
A: 是的，设置存储在浏览器本地。清除缓存后需要重新设置。

### Q4: 可以上传什么样的图片作为背景？
A: 支持 JPG、PNG、GIF 格式，最大 2MB。建议使用高清、色彩简洁的图片。

### Q5: 深色模式对所有主题都有效吗？
A: 深色模式本身就是一个主题，选择"深色模式"即应用深色配色。

### Q6: 为什么有些内容有链接，有些没有？
A: 这取决于内容发布者是否选择添加链接。链接是可选项。

### Q7: 如何批量添加链接到已有内容？
A: 目前需要在管理后台逐个编辑。可以通过直接编辑 JSON 文件批量添加。

## 开发者说明

### 主题配置文件
位置：`data/themes.json`

```json
{
  "themes": [...],
  "defaultTheme": "default",
  "backgrounds": [...],
  "defaultBackground": "default-bg"
}
```

### 添加新主题
在 `themes.json` 的 `themes` 数组中添加：

```json
{
  "id": "custom",
  "name": "自定义主题",
  "description": "描述",
  "primaryColor": "#xxx",
  "secondaryColor": "#xxx",
  "accentColor": "#xxx",
  "bgColor": "#xxx",
  "textColor": "#xxx",
  "lightTextColor": "#xxx",
  "bgLight": "#xxx"
}
```

### 添加新背景
在 `themes.json` 的 `backgrounds` 数组中添加：

```json
{
  "id": "custom-bg",
  "name": "自定义背景",
  "description": "描述",
  "type": "gradient|image|solid",
  "value": "渐变|图片URL|颜色值"
}
```

### 超链接数据结构
所有内容类型支持以下字段：

```json
{
  "hasLink": true,
  "linkUrl": "https://...",
  "linkText": "点击查看"
}
```
