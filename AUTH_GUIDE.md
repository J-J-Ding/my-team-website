## 登录指南

### 访问登录页面
1. 打开网站首页
2. 点击"成员登录"按钮
3. 或直接访问 `pages/login.html`

### 登录步骤
1. 输入用户名和密码
2. 点击"登录"按钮
3. 系统会自动跳转到对应的控制台
   - 管理员 → 管理后台
   - 成员 → 个人中心

### 退出登录
点击右上角的"退出登录"按钮

## 成员使用指南

### 进入个人中心
登录后自动跳转到个人中心 (`pages/dashboard.html`)

### 功能模块

#### 1. 👤 个人资料
**功能**: 更新基本信息
- **姓名**: 显示在团队卡片上
- **职位**: 您在团队中的职位
- **个人描述**: 简短介绍
- **工作经验**: 工作年限
- **头像**: 输入姓氏作为头像

**步骤**:
1. 点击左侧"个人资料"
2. 填写或修改信息
3. 点击"保存更改"

**说明**:
- 带 `*` 为必填项
- 所有更改实时预览

#### 2. 💼 技能管理
**功能**: 添加您的专业技能
- 每行一个技能
- 最多 8 个技能
- 实时预览效果

**示例输入**:
```
React
Vue
TypeScript
Node.js
```

**步骤**:
1. 点击左侧"技能管理"
2. 在文本框中输入技能（每行一个）
3. 实时查看预览
4. 点击"保存技能"

#### 3. 🔗 社交链接
**功能**: 添加社交媒体链接
- LinkedIn
- GitHub
- Twitter（可选）

**步骤**:
1. 点击左侧"社交链接"
2. 输入完整链接地址
3. 点击"保存链接"

**说明**:
- 链接格式：`https://...`
- 可留空

#### 4. 👁️ 预览效果
**功能**: 查看在团队页面的显示效果
- 实时预览团队卡片
- 点击"在团队页面查看"跳转

**步骤**:
1. 点击左侧"预览效果"
2. 查看卡片预览
3. 点击"刷新预览"更新
4. 点击"在团队页面查看"查看实际效果

## 常见问题

### Q1: 忘记密码怎么办？
A: 目前请联系管理员重置密码。默认成员密码为 `123456`。

### Q2: 成员修改的信息会实时更新到网站吗？
A: 会。修改个人资料后，网站团队页面会立即更新显示。

### Q3: 管理员可以修改成员的密码吗？
A: 可以，管理员可以编辑所有成员的信息，包括密码（通过直接编辑数据文件）。

### Q4: 如何创建新成员账号？
A: 管理员在"团队成员管理"中添加成员后，需要在 `data/auth/users.json` 中创建对应的登录账号。

### Q5: 数据保存在哪里？
A:
- 成员数据: `data/team/members.json`
- 服务数据: `data/services/services.json`
- 用户账号: `data/auth/users.json`
- 公司信息: `data/about/company.json`
- 联系信息: `data/contact/info.json`

### Q6: 为什么修改后没有生效？
A:
1. 确保点击了"保存"按钮
2. 刷新浏览器页面
3. 检查是否有 JavaScript 错误

### Q7: 支持批量导入成员吗？
A: 当前版本不支持批量导入，需要逐个添加。可以手动编辑 JSON 文件批量添加。

### Q8: 如何备份数据？
A: 定期备份 `data/` 文件夹下的所有 JSON 文件。

## 技术说明

### 数据存储
所有数据存储在 JSON 文件中，部署到 GitHub Pages 后是静态的。

### 数据更新限制
由于是静态网站，**成员修改的数据需要手动同步到 GitHub**:
1. 管理员或成员在本地修改
2. 将修改的 JSON 文件提交到 Git
3. 推送到 GitHub
4. GitHub Actions 自动部署

### 实时更新的实现
- 使用 JavaScript 从 JSON 文件加载数据
- 页面渲染时动态读取数据
- 无需后端服务器

---

## 安全建议

1. **修改默认密码**: 首次使用后修改所有默认密码
2. **数据备份**: 定期备份 `data/` 文件夹
3. **权限控制**: 限制 JSON 文件的直接访问
4. **HTTPS**: 生产环境使用 HTTPS
